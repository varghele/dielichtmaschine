# Demo rigs and shows

Reproducible, standard stage-lighting rigs for testing and as starter
projects. Each is a `Configuration` project file (`.lms`, YAML content) under `rigs/`, generated
deterministically by [`generate_rigs.py`](generate_rigs.py). Ready-to-play
[demo *shows*](#demo-shows) are layered on top of them.

Open one via **File → Open Configuration** (or load it in a test).

## Why these load anywhere

The rigs reference **only** the fixtures bundled in `custom_fixtures/`, so
they patch, classify, and render identically on any machine — no dependency
on the user's QLC+ install. (The stock QLC+ *Generic* manufacturer folder has
no moving heads, beams, strobes, or pixel bars, so it can't express these
layouts at all.)

## The rigs

| File | Fixtures | Shape | Exercises |
|------|---------:|-------|-----------|
| `club_band.lms` | 9 | 4 front PARs, 2 rear washes, 2 movers, 1 blinder | Smallest viable rig; fast load path |
| `band_midsize.lms` | 21 | Front/back PARs, spots, moving wash, LED bars, blinders, matrix | Every `lighting_role`; all four sublane types |
| `festival_mainstage.lms` | 60 | Front-wash / mid-spot / back-beam truss rows, floor beams, blinders, matrix, side towers | Rows-of-movers look; multi-universe (3) export scale; visualizer archetype breadth |
| `dj_edm.lms` | 20 | Beam array, moving wash, pixel bars, matrix, strobes | Movement-centric; matrix/beam chassis paths |
| `theatre_static.lms` | 16 | Front PAR wash, rear cyc wash, side booms | Zero-movement export + capability detection |

## Coordinate convention

`X` left→right `[-W/2, +W/2]`; `Y` depth, **centred** (`y<0` downstage/front,
`y>0` upstage/back); `Z` height in metres. Truss fixtures sit overhead
(`z` ≈ 3–8 m), floor packages near `z≈0.2`.

## Regenerating

```sh
python -m demos.generate_rigs
```

Edit the `RIGS` table in `generate_rigs.py` to add or reshape a rig. Modes and
channel counts are read live from the `.qxf` files, so DMX addressing and
`available_modes` stay correct automatically. The generator auto-rolls to a new
universe at the 512-channel boundary.

# Demo shows

Ready-to-play light shows layered on the rigs above — one per rig under
[`shows/`](shows/), each the rig plus an autogenerated `Demo` show driven by a
bundled audio clip. A new user can open a populated project and hit play instead
of landing in an empty one.

Open via **File → Open Configuration**, then pick the `Demo` show in the Shows tab.

### The clip

[`shows/audiofiles/monsters_demo.ogg`](shows/audiofiles/) is a ~24 s
build-into-chorus excerpt (prechorus + chorus, 192 BPM) of the reference track —
a small compressed OGG, so it barely weighs on the repo. The full-length track
and the original hand-made touring show it came from live under
[`reference/`](reference/) (see the legacy → timeline conversion in
[`utils/legacy_show_converter.py`](../utils/legacy_show_converter.py)).

### How they're made

[`generate_shows.py`](generate_shows.py) runs the autogen pipeline against the
clip, but takes the **song structure** (parts, BPM, bar counts) from the real
hand-made show rather than a generic template, so the sections line up with the
music. Output is deterministic (fixed seed) and is ordinary timeline blocks —
open a show and edit it freely.

```sh
python -m demos.generate_shows shows/audiofiles/monsters_demo.ogg \
  --structure-from reference/SBD_touring.yaml \
  --structure-show SBD_monsters_in_my_head --structure-slice 4:6
```

`--structure-slice 4:6` keeps just the prechorus + chorus parts (indices into
the reference show) to match the short clip; drop it to structure a full-length
show, or drop `--structure-from` entirely for a generic intro/verse/chorus
template sized to the clip.

### Rendering README media

Turn any demo show into clean stills / an animated GIF (or a full-res MP4) with
[`generate_media.py`](generate_media.py) — headless, no window needed:

```sh
python -m demos.generate_media dj_edm               # clean stills + GIF (default)
python -m demos.generate_media dj_edm --mp4         # full-res video (needs imageio-ffmpeg)
python -m demos.generate_media dj_edm --show Demo --cameras "Front,Wide"
```

Output lands in `media/<rig>/`; MP4s are git-ignored (regenerate on demand),
stills and GIFs are small enough to commit.
